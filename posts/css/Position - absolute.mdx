## 서론

최근 개발을 하며 스스로 잘 알고, 잘 사용하고 있다고 생각한 position absolute 를 사용하다, 생각보다 해당 속성에 대해 잘 모른다는 사실을 알게 됐다. modal을 제작하는 중 absolute를 적용하는 과정에서 문제가 발생했고 이를 계기로 삼아 absolute에 대해 자세히 알아보고자 한다.

## 본론

### Position : absolute

본론에 들어가기에 앞서, 우선 `position : absolute`의 정의를 확인하고자 한다. [mds](https://developer.mozilla.org/ko/docs/Web/CSS/position)에서 상세 내용을 확인해보자.

>
요소를 일반적인 문서 흐름에서 제거하고, 페이지 레이아웃에 공간도 배정하지 않습니다. 대신 가장 가까운 위치 지정 조상 요소에 대해 상대적으로 배치합니다. 단, 조상 중 위치 지정 요소가 없다면 초기 [컨테이닝 블록](https://developer.mozilla.org/ko/docs/Web/CSS/CSS_display/Containing_block)을 기준으로 삼습니다. 최종 위치는 `top`, `right`, `bottom`, `left` 값이 지정합니다.
[`z-index`](https://developer.mozilla.org/ko/docs/Web/CSS/z-index)의 값이 `auto`가 아니라면 새로운 [쌓임 맥락](https://developer.mozilla.org/ko/docs/Web/CSS/CSS_positioned_layout/Stacking_context)을 생성합니다. 절대 위치 지정 요소의 바깥 여백은 서로 [상쇄](https://developer.mozilla.org/ko/docs/Web/CSS/CSS_box_model/Mastering_margin_collapsing)되지 않습니다.

결국 `position : absolute`로 선언된 요소는 일반적인 문서 흐름에서 제거 되어, 특정 조상 요소에 대해서 **상대적으로** 배치된다는 것을 의미한다. 

이 때, 많은 사람들은 absolute로 선언된 요소의 부모 요소에 `position : relative`을 으레 선언하는 것을 알고 있을 것이다. 

당연하지만 이는 전통이나 예법에 따른 것이 아니라, `absolute`의 배치 규칙을 위해서 이다.


### Absolute 의 기준 요소

상기한 대로, absolute는 **가장 가까운 위치 지정 조상 요소** 를 기준으로 상대적으로 배치된다. 아니 이게 무슨 소리일까?  말로 하면 참 오묘한 말이 되지만, 마찬가지로 mdn에서는 명확한 기준을 설명하고 있다.  

>`position` **속성이 `absolute`**인 경우, 컨테이닝 블록은 `position` 속성 값이 `static`이 아닌(`fixed`, `absolute`, `relative`, `sticky`) 가장 가까운 조상의 내부 여백 영역입니다

즉, absolute의 부모 요소는 반드시 relative여야 하는 것은 아니고, static 이 아닌 요소를 기준으로 배치된다. 또한, 부모 요소가 static인 경우 static이 아닌 요소를 만날 때 까지 추적해 올라간다. 기본적으로 요소는 position : static 이므로, 별도의 설정을 하지 않으면 body 까지 거슬러 올라가 버린다.

때문에 부모 요소에 position : relative 를 선언해주는 것이 관례처럼 자리잡은 것이다.  
만일 absolute가 의도한대로 동작하지 않는다면, 우선 확인해봐야 하는 부분이다.  

또 중요하게 보아야 하는 것이, 페이지 레이아웃에 공간이 배정되지 않는 다는 점이다. 그렇다면 이런 경우를 생각해 볼 수 있겠다. 

```HTML
<!-- 배경요소 -->
<div style="
        position: fixed;
        left: 0;
        top: 0;
        width: 100%;
        height: 100%;
        background-color: red;
      "> 
	<!-- 모달 내부 컨텐츠 -->
	<div style="
          position: absolute;
          width: 120px;
          left: 50%;
          top: 50%;
          background-color: yellow;
        ">
		<!-- 컨텐츠... -->
	</div>
</div>
```
간단한 modal을 만드는 경우로, 주변 배경을 어둡게 하기 위한 배경 요소는 고정된 위치로 배치되어야 한다. 하지만 모달 내부가 너무 길어진 경우, overflow 값을 조절하고 싶어질 수 있다. 내부 요소를 스크롤 하고 싶다면 어디서 overflow를 핸들링 해야 할 까?
