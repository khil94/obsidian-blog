현재 : #전적페이지 에서는 주소의 소환사명/태그명을 조합하여 api를 호출하여 전적을 가져오고 있음.

문제 : - 태그가 추가됨에 따라, #랭킹페이지 와 전적상세보기에서 다른 플레이어를 클릭하여 전적페이지로 들어가는것이 불가능하게 되었음. 이를 해결하기 위해 사용자의 uid를 이용하여 전적을 확인하는 api가 출시됨. - 하지만, 기존 전적페이지는 url의 정보를 이용해 api를 호출하는 방식이므로 해당 방식은 불가능함. - 로직을 바꾸어야 함.

해결법 : - url에 더 많은 정보를 주어서 변경? - name? 혹 id? 이런 식으로 url params에 주어진 정보를 확인, 이에 대응하여 api를 호출하는 방안 - 애초에 페이지로 진입시키는 곳에서 정보를 던져주는 방식 - 이러면 새로고침이나 url을 직접 입력하여 들어올 시 문제가 발생할 거 같은데?
